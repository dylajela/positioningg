Definition


The CSS position property defines the position of an element in a document. This property works with the left, right, top, bottom and z-index properties to determine the final position of an element on a page.

There are five values the position property can take:

Static


Do not be affected by the left, right, top, bottom, and z-index properties.

Relative


When assigned position: relative, an element follows the render flow of the page, but will be shifted relative to its initial position.


Surrounding elements won't be affected, but there will be soace repositioned by the left, right, top, and bottom properties.

Absolute


With position: absolute, an element ignores the normal document flow. However, instead of being positioned relative to the viewport, it’s positioned relative to the nearest positioned ancestor (the default)

Fixed


Elements with position: fixed do not adhere to the normal render flow of the document. Instead, fixed elements are positioned relative to the viewport


Fixed elements do not move when the user scrolls. Can use the top, right, bottom, and left properties to set the fixed element's final position

Sticky


Elements with position: sticky are positioned depending on the user’s scroll. Depending on how far the user has scrolled down the page, a sticky element behaves like a relative element until the viewport meets a specified position. Then it becomes fixed in a spot and appears to “stick” to the page as the user scrolls.
